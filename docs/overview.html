🌅 The product in one paragraph

You sign in with Google, tell Thedi your research interests (keywords + sources), and every morning at 7 AM local you get a personalized email with 10 ranked items from arxiv + HN + X. Each item has a one-sentence "why it matters for you" angle. Click "Tell Thedi what missed" in any email → land on a chat page with the scout pre-seeded with two of today's items; type free-text critique; Qwen extracts structured preferences; tomorrow's critic reads them and drops the items you didn't want.

The rank diff is visible on a replay page — you can see exactly which papers the selector picked initially, what the critic said about them, and which the refiner swapped in. It's not a black box.

🔧 The polishing loop (MAKER-lite)

One Butterbase function — scout-polish-now — runs the whole chain synchronously in ~13 seconds:

arxiv + HN (keyword-filtered) │ ▼ INGEST 25–50 raw items │ ▼ SELECT Qwen picks top 10 + per-item angle │ ▼ CRITIC Qwen reviews: "is this serving the user?" │ ├── (accepted) ────────────┐ │ │ └── (rejected) │ ▼ │ REFINE re-rank │ │ │ └─────────┬───────────┘ ▼ RED-FLAG URL allowlist + title sanity │ ▼ FINALIZE write findings + digest row │ ▼ DELIVER Resend email to user │ ▼ user clicks "Tell Thedi what missed" │ ▼ extracted_prefs → tomorrow's CRITIC

Every step writes to audit_log with tokens, ms, and status — so any digest is fully debuggable.

🎭 The meta-process — RALF on the design itself

Before writing a line of code, we ran 3 rounds of RALF (Refine-And-Loop-Forward) on the Thedi design doc. Each round: 3 parallel Claude critics with different lenses (Product/UX, Engineering, Delivery/Ops → Security, Cost, Edge Cases → YAGNI, Demo Narrative, Butterbase Feasibility) proposed changes; a judge synthesized. 27 proposals accepted across 9 lenses.

What each RALF round caught

Round 1
Idempotency (UNIQUE on user_id+run_date), failure ceilings, email subject templated from top finding, feedback chat pre-seeded, 30-second landing page, URL allowlist for red-flag
Round 2
Token-gated share URLs, audit_log PII scrubbing + 14-day TTL, admin token model, critic token budget (–60% spend), golden-set eval, temperature=0, DST backfill, Unicode keyword quoting, welcome digest on first signup
Round 3
(load-bearing) → 1-pass critic (Sonnet → Haiku), 9→6 tables, SPA instead of per-digest HTML (Butterbase is zip→Pages), step-function state machine (Butterbase caps functions at 5 min), pre-run at T-10 for demo, replay page with critic stream, live re-rank preview button

Artifacts: docs/superpowers/specs/2026-04-19-thedi-design.md (v3 final) + docs/superpowers/ralf/round-{1,2,3}-changelog.md.

🧠 What Thedi does that nobody else does

From the research — see docs/features/research-scout-landscape.md:

  1. Multi-agent selector → critic → refiner chain on a daily cadence. Elicit runs a pipeline but per-query; Smol.AI runs agents but non-personalized; Perplexity/OpenAI Deep Research are one-shot. No competitor ships a daily-push product with an adversarial critic + red-flag reviewer per item.
  2. Structured "Not useful? Tell me why" feedback that feeds the next day's critic prompt. Semantic Scholar has "not relevant" but it only reorders rankings — it doesn't change tomorrow's selector prompt. Free-text reason extraction closing the loop is genuinely novel in this cohort.
  3. arXiv + HN + X fused and critiqued together into a single 10-item digest at 7 AM local. Emergent Mind uses all three but only for ranking arxiv; Smol.AI digests Discord/Reddit/X but not arxiv first-class; no one else fuses all three with per-item critic scoring + personalization.
  4. Rank-diff visible in the product (replay page shows initial → final movement). Unique — no competitor exposes their agent's reasoning this way.

💬 The moment that sold it

Thedi's critic to Ashish's digest
"Items #8, #9, and #10 are too tangential to the user's core focus on agentic AI for creative/video applications; #8 discusses RL sim2real transfer, #9 covers LLM problem-solving generalization, and #10 focuses on event-camera hardware, none of which directly support building agentic tools for video generation. Replace these with papers on multi-agent collaboration frameworks, LLM-based video editing agents, or benchmarks for creative task evaluation in agentic systems."

This is the visible reasoning that sets Thedi apart from "one-size-fits-all" AI digest newsletters. The critic argues with itself, and you get to watch.
